# 田中祐一AI 添削出力アーキテクチャ

この文書は、田中祐一AIの添削モードで生成する静的レポートのページ構造、リンク生成ルール、素材タイプ別の表示仕様を固定するためのマニュアルです。

目的は、AIや担当者が変わっても「ビジュアルレポート」「テキストレポート」「原本素材集」の動線が崩れないようにすることです。

## 基本方針

- 添削出力は「ファネルレポートポータル」を入口にする
- 主要成果物は、ビジュアルレポート、テキストレポート、原本素材集の3種類
- 左サイドバーは全ページ共通で表示する
- 手打ちリンクを増やさず、素材IDとページ種別からURLを決める
- 原本素材集には評価や改善案を入れない
- ビジュアルレポートにはキャプチャ、網掛け、番号付きフィードバックを入れる
- テキストレポートはマーケター向けに、白基調の読み物として1カラム本文で構成する

## 正しいページ構造

標準構造は以下です。

```text
<report-slug>/
├─ index.html                         # ファネルレポートポータル
├─ visual-report.html                 # ビジュアル全体レポート
├─ text-report.html                   # テキスト全体レポート
├─ visual/
│  ├─ optin-lp.html                   # ビジュアル: オプトインLP
│  ├─ thanks.html                     # ビジュアル: サンキューページ
│  ├─ stepmail.html                   # ビジュアル: ステップメール全体/一覧
│  ├─ line-step.html                  # ビジュアル: ステップLINE全体/一覧
│  ├─ live-day1.html                  # ビジュアル: ライブ1
│  ├─ ...
│  ├─ materials.html                  # 原本素材集トップ
│  └─ source/                         # 原本MD
└─ text/
   ├─ optin-lp.html                   # テキスト: オプトインLP
   ├─ thanks.html                     # テキスト: サンキューページ
   ├─ stepmail.html                   # テキスト: ステップメール全体/一覧
   ├─ line-step.html                  # テキスト: ステップLINE全体/一覧
   ├─ live-day1.html                  # テキスト: ライブ1
   └─ ...
```

暫定的に `text-report.html` へ集約する場合でも、サイドバーの素材リンクをページ内アンカーだけにし続けるのは最終仕様ではありません。最終仕様では、素材ごとのテキストページを作ります。

## URL生成ルール

素材ごとに、必ず安定した `materialId` を持たせます。

| 素材 | materialId | ビジュアルURL | テキストURL | 素材集URL |
|---|---|---|---|---|
| オプトインLP | `optin-lp` | `visual/optin-lp.html` | `text/optin-lp.html` | `visual/materials.html?item=optin-lp` |
| サンキューページ | `thanks` | `visual/thanks.html` | `text/thanks.html` | `visual/materials.html?item=thanks` |
| ステップメール | `stepmail` | `visual/stepmail.html` | `text/stepmail.html` | `visual/materials.html?group=stepmail` |
| ステップLINE | `line-step` | `visual/line-step.html` | `text/line-step.html` | `visual/materials.html?group=line-step` |
| ライブ1 | `live-day1` | `visual/live-day1.html` | `text/live-day1.html` | `visual/materials.html?item=live-day1` |
| 個別説明会ページ | `consult-lp` | `visual/consult-lp.html` | `text/consult-lp.html` | `visual/materials.html?item=consult-lp` |

### 禁止ルール

- テキストレポートの素材ナビから `visual/<materialId>.html` へ飛ばさない
- 素材集の素材ナビから評価レポートへ飛ばさない
- ビジュアルレポートの右側フィードバック項目を、左サイドバーに重複表示しない
- 「第1層」「第2層」「第3層」など、作業指示用の内部用語を顧客向けHTMLに出さない

## 左サイドバーの固定ルール

全ページに以下のサイドバーを固定します。

```text
レポート
  ファネルレポートポータル
  ビジュアルレポート
  テキストレポート
  原本素材集

ファネル素材
  オプトインLP
  サンキューページ
  ステップメール
  ステップLINE
  ライブ1
  ライブ2
  ライブ3
  ライブ4
  ライブ5
  個別説明会ページ
  体験セミナー誘導ページ
  追加セミナー1
  追加セミナー2
  オープンチャット
```

ただし、サイドバーのリンク先は、今いるページ種別によって変えます。

| 現在地 | 素材リンクの遷移先 |
|---|---|
| ポータル | 原則ビジュアルページ。必要なら素材概要へ |
| ビジュアルレポート | `visual/<materialId>.html` |
| テキストレポート | `text/<materialId>.html` |
| 原本素材集 | `visual/materials.html?item=<materialId>` |

つまり、同じ「オプトインLP」という表示でも、ページ種別によってリンク先を切り替えます。

## 3種類の役割分担

### ファネルレポートポータル

役割は入口です。

- ビジュアルレポート、テキストレポート、原本素材集への3つの大きなボタンを置く
- 「3点セット」という内部呼称は使わない
- 表示名は「ファネルレポートポータル」または「添削出力ポータル」とする
- 素材ごとの細かい評価は載せない

### ビジュアルレポート

役割は、素材とフィードバックの対応を視覚的に確認することです。

- 左側に素材キャプチャ、動画代表フレーム、または書き起こし本文を置く
- 右側に番号順のフィードバックを置く
- 指摘は「優先度 高・中・小」で分類する
- フィードバックはアコーディオンで開閉できる
- 文章素材では対象箇所に網掛けを入れる
- 動画素材では冒頭キャプチャを置き、書き起こしに対する評価と分ける
- 良い点も書く。悪い点だけの指摘にしない

### テキストレポート

役割は、マーケターがファネル全体と改善論点を読み込むことです。

- 白基調にする
- 左サイドバーは残す
- 右側本文は1カラムで上から下へ読める構造にする
- ビジュアル用のキャプチャや2カラム比較は入れない
- 用語整理、素材対応表、ファネル全体像、全体所感、優先度内訳、素材別フィードバックを掲載する
- 素材ごとの詳細ページを作る場合も、本文は1カラムを維持する

### 原本素材集

役割は、添削対象の原本を見返すことです。

- 評価、改善案、キャプチャ評価を入れない
- 冒頭に原本URLを置く
- 動画は可能なら埋め込み、難しければ原本URLを明示する
- 動画書き起こしには10分ごとのタイムスタンプを入れる
- LPは実際のURL、保存済みMD、画面テキスト全文を置く
- ステップメール/LINEは一覧から1段深く入り、各通の本文を確認できるようにする
- 2カラムで比較するのはビジュアルレポートだけ。素材集は原本確認を優先する

## 素材タイプ別チェック観点

| 素材タイプ | ビジュアルレポート | テキストレポート | 原本素材集 |
|---|---|---|---|
| LP/サンクス/説明会LP | 縦長キャプチャ + 番号付き指摘 | 役割、導線、CTA、改善案 | URL + 保存MD + 本文 |
| 動画/ライブ | 冒頭キャプチャ + 書き起こし網掛け + 映り方評価 | 構成、訴求、離脱ポイント、CTA | 動画URL/埋め込み + 10分刻み書き起こし |
| ステップメール | 全体ページ + 各通の指摘 | フェーズ設計、件名、接続、CTA | 一覧 + 各通本文 |
| ステップLINE | 全体ページ + 各通の指摘 | 短文導線、通知文脈、CTA | 一覧 + 各通本文 |
| オープンチャット | 固定投稿、投稿導線、質問設計 | 参加者行動、質問誘導、運用改善 | ログ本文 |

## ファネル形状別チェック観点

### LP単体添削

- LPキャプチャがあるか
- ファーストビュー、ベネフィット、実績、CTA、保証、FAQを確認したか
- 素材集にはLP原本だけがあるか

### ステップ配信中心

- ステップメール/LINEが1段深い構造になっているか
- 一覧から各通へ移動できるか
- 全体所感ページと各通フィードバックページが分かれているか

### 動画ローンチ/ライブ型

- 各動画のURLまたは埋め込みがあるか
- 冒頭3秒/5秒/10秒などの代表フレームがあるか
- 書き起こしにタイムスタンプがあるか
- 動画ごとのCTAと次回予告が評価されているか

### 統合ファネル分析

- 全体ページに素材ごとの指摘件数と優先度内訳があるか
- LP、サンクス、配信、ライブ、説明会、締切後導線が横断的に見られるか
- 各素材ページへ迷わず戻れるか

## 生成後の必須チェック

自動チェックで最低限確認します。

```bash
node scripts/check-report-output.mjs <report-dir> \
  --public-base https://tanakayuc.github.io/static-pages/<report-dir>
```

目視チェックでは、最低でも以下を開きます。

- `index.html`
- `visual-report.html`
- `text-report.html`
- `visual/materials.html`
- LP系の代表ページ
- 動画系の代表ページ
- ステップメール/LINEの一覧ページ
- ステップメール/LINEの個別ページ

## 完了条件

- ページ構造がこの文書と一致している
- 主要リンクがページ種別ごとの正しい遷移先になっている
- 原本素材集に評価が混ざっていない
- テキストレポートからビジュアル素材ページへ誤遷移しない
- ビジュアルレポートの右側指摘が、左サイドバーに重複していない
- 公開URLで旧表示と旧リンクが消えている
