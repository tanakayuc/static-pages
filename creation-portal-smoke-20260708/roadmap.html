第1章 事前設計

プロモーションを始める前に、出口商品、販売ファネル、販売モデル、KPIを仮決めする。教育グループとライブ日数はターゲット・コンセプト確定後に決める。

  1. 1-1販売したい商品を決める

    販売する商品、価格、期間、販売本数を仮決めする。

    OUTPUT: 出口商品リスト / 未着手
  2. 1-2販売ファネルを決める

    今回採用するファネル種別をチャレンジローンチとして仮決めする。

    OUTPUT: 販売ファネル / 未着手
  3. 1-3販売モデルを決める

    WEB直接販売、個別相談、体験セミナー経由、セミナーセールスのどれで販売するかを仮決めする。

    OUTPUT: 販売モデル / 未着手
  4. 1-4目標KPIの設定

    仮売上と商品単価から必要成約数、必要申込数、必要リスト数、必要アクセス数を逆算する。

    OUTPUT: KPI逆算メモ / 未着手

第2章 コンセプト設計

背景情報、ターゲット、リサーチ、ポジショニング、コンセプト、プロフィールを作る。

  1. 2-1事前情報収集（背景情報）

    本人情報、実績、背景、判断材料を集める。

    OUTPUT: プロフィールシート / 未着手
  2. 2-2ターゲットシート作成

    見込み客の悩み、願望、動く条件を整理する。

    OUTPUT: ターゲットシート / 未着手
  3. 2-3ライバル情報の収集

    同業ライバルの訴求、媒体、商品、強みを整理する。

    OUTPUT: ライバル分析シート / 未着手
  4. 2-4空いている訴求の特定

    ライバルの強訴求から、ついていけない人と狙える余白を整理する。

    OUTPUT: 空き訴求リスト / 未着手
  5. 2-5ポジショニングの作成

    ターゲット、ライバル、コンテンツホルダーの強みをつなぎ、狙う立ち位置を言語化する。

    OUTPUT: ポジショニングシート / 未着手
  6. 2-6プロモーションコンセプト

    LP、配信、台本、販売ページへ展開する中核コピーを決める。

    OUTPUT: プロモコンセプト / 未着手
  7. 2-7プロモヘッダーバナーの作成

    プロモーションの見た目に使うヘッドコピーと画像方向を決める。

    OUTPUT: ヘッド指示書 / 未着手
  8. 2-8プロフィール作成（ブラッシュアップ）

    ポジショニングに合わせて信頼形成用プロフィールを整える。

    OUTPUT: プロフィール文 / 未着手

第3章 オファー構築

提供内容、期間、価格、特典、保証をオファーシートへまとめる。

  1. 3-1本命商品オファー

    商品名、特徴、サポート期間、価格、特典、保証を整理する。

    OUTPUT: オファーシート / 未着手

第4章 コンテンツ設計

ターゲットとコンセプトを踏まえて、教育グループ、チャレンジ日数、5日間の内容、課題フォーム、コアストーリー、登録直後の案内素材を決める。

  1. 4-1教育グループを決める

    ターゲットとコンセプトを踏まえ、参加者をどこに集めるかを決める。

    OUTPUT: 教育グループ方針 / 未着手
  2. 4-2チャレンジ日数・ライブ回数を決める

    2チャレ / 3チャレ / 4チャレ / 5チャレのどれで進めるか、ライブ本数を決める。

    OUTPUT: チャレンジ日数・ライブ本数 / 未着手
  3. 4-3お客様の声ファイル生成

    既存の感想、実績、体験談を素材として使える形にする。

    OUTPUT: お客様の声リスト / 未着手
  4. 4-43日間コンテンツ設計

    3日分の価値提供テーマと販売接続を設計する。

    OUTPUT: 3日間テーマ一覧 / 未着手
  5. 4-5チャレンジ課題フォームの整理

    3日分の課題と提出フォームを整理する。

    OUTPUT: 課題フォームメモ / 未着手
  6. 4-6スケジュール決め

    集客開始、ライブ日、販売開始、締切のカレンダーを決める。

    OUTPUT: プロモ全体カレンダー / 未着手
  7. 4-7ライブ1コアストーリー

    初回ライブで語るコアストーリーを作る。

    OUTPUT: コアストーリーシート / 未着手
  8. 4-8サンキューページVSLシナリオ作成

    オプトイン後に見せるVSLのシナリオを作る。

    OUTPUT: サンキューページVSLシナリオ / 未着手
  9. 4-9登録直後案内素材を用意する

    登録直後に見せる案内、動画、グループ誘導の素材を用意する。

    OUTPUT: 登録直後案内素材 / 未着手

第5章 オプトインLPの作成

LP原稿、画像指示、実装指示、サンキューページ原稿を作る。

  1. 5-1オプトインLP ライティング

    ファーストビュー、共感、ベネフィット、参加理由、CTAを原稿化する。

    OUTPUT: オプトインLP原稿 / 未着手
  2. 5-2画像指示書の作成

    LPに必要な画像、ヘッド、CTA、スマホ表示、参考サンプルをまとめる。

    OUTPUT: LPヘッドデザイン指示書 / 未着手
  3. 5-3画像素材の作成

    必要な場合のみ、LPで使う画像素材を作る。

    OUTPUT: LP用画像素材 / 未着手
  4. 5-4HTMLコードの生成と公開

    実装担当へ渡せるHTML化指示と公開先メモを作る。

    OUTPUT: LP実装指示書 / 未着手
  5. 5-5サンキューページライティング

    VSL視聴を前提にしたサンキューページの原稿を書く。

    OUTPUT: サンキューページ原稿 / 未着手
  6. 5-6サンキューページ作成

    VSL埋め込み付きのサンキューページを実装する。

    OUTPUT: サンキューページ実装指示書 / 未着手

第6章 集客素材の作成

教育グループ、固定投稿、自動返信、紹介文、広告素材を用意する。

  1. 6-1教育グループ（LINEオープンチャット）作成

    参加場所を作り、登録後に案内できる状態にする。

    OUTPUT: 教育グループリンク / 未着手
  2. 6-2LINEオープンチャットの固定メッセージを整える

    参加者が最初に見る固定投稿を作る。

    OUTPUT: 固定投稿 / 未着手
  3. 6-3LINEオープンチャット事前案内の整備

    ライブ開始前までに送る事前案内文を作る。

    OUTPUT: 事前案内文 / 未着手
  4. 6-4オプト後自動返信メール

    登録直後に教育グループへ戻すメール文面を作る。

    OUTPUT: 自動返信メール文面 / 未着手
  5. 6-5セットアップライティング

    メルマガやSNSで登録ページへ送る紹介文を作る。

    OUTPUT: 紹介文章 / 未着手
  6. 6-6セットアップ用画像生成

    必要な場合のみ、投稿や紹介に使う画像方向を作る。

    OUTPUT: 投稿用画像指示書 / 未着手

Day1〜Day5の台本と課題フォームを作る。

  1. 7-1ライブ1シナリオ作成

  4. 7-4チャレンジ課題のアウトプットフォーム

第8章 プロモ素材の作成

体験セミナー申込ページ、返信、個別相談への接続素材を作る。

  1. 8-1リマインドメール

    教育グループ登録誘導とライブ前リマインドを作る。

    OUTPUT: リマインド文面 / 未着手
  2. 8-2LINEオープンチャット投稿文章

    期間中に送る固定投稿と通常配信を時系列で作る。

    OUTPUT: LINE投稿文 / 未着手
  3. 8-3体験セミナー申込ページ

    体験セミナーへ申し込む理由と参加後の流れを伝えるページを作る。

    OUTPUT: 体験セミナーLP原稿 / 未着手
  4. 8-4体験セミナー自動返信

    申込完了、日時、参加方法、事前準備を伝える自動返信を作る。

    OUTPUT: 体験セミナー自動返信メール / 未着手
  5. 8-5体験セミナーサンキューページ

    体験セミナー申込後のサンキューページを作る。

    OUTPUT: 体験セミナーサンキューページ原稿 / 未着手
  6. 8-6個別相談申込導線

    体験セミナー後に個別相談へ進む導線を作る。

    OUTPUT: 個別相談導線原稿 / 未着手

第9章 本番運用

公開前の最終準備、集客実施、ライブ運用、販売、法務、改善で確認する項目。

  1. 9-1集客の実施

    用意した紹介文、投稿、広告素材から登録ページへ送る。

    OUTPUT: 目標オプト数 / 未着手
  2. 9-2ライブの実施

    Day1〜Day5の台本に沿ってライブ、課題、質問対応を行う。

    OUTPUT: ライブ実施ログ / 未着手
  3. 9-3体験セミナー販売スライド

    セミナーで価値提供し、個別相談へ自然につなげるスライドを作る。

    OUTPUT: 体験セミナースライド / 未着手
  4. 9-4個別相談セールストーク

    個別相談で使う質問、反論対応、クロージングの流れを作る。

    OUTPUT: 個別相談クロージング台本 / 未着手
  5. 9-5特商法・プライバシーポリシー

    販売条件に合わせて必要ページを確認する。

    OUTPUT: 特商法/プライバシーポリシー / 未着手
  6. 9-6法定書面

    該当する場合のみ概要書面と契約書面を用意する。

    OUTPUT: 概要書面/契約書面 / 未着手
  7. 9-7セールス改善（次回プロモへ）

    成約データ、離脱データ、録画から次回の改善点をまとめる。

    OUTPUT: 改善レポート / 未着手
