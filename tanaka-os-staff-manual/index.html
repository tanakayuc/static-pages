1田中祐一OSとは

受講生が 自分の原稿（LP・動画台本・オファー・セールスレター 等）を AI に貼って、田中祐一として添削してもらえるツールです。

裏側では、田中祐一が積み上げてきた 「11観点 × ジャンル別の固有論点 × 田中祐一メソッド核心原則」のナレッジを毎回プロンプトに食わせています。AI（Claude / ChatGPT / Gemini）は単に文章を整えるのではなく、田中祐一の判断軸で添削します。

ポイント: 受講生が使う AI（Claude / ChatGPT / Gemini）は受講生自身のサブスクで動きます。田中さん側の AI コストは原則ゼロ。受講生は好きなモデルで添削を受けられます。

4Web UI で添削する

これが受講生のメイン動線です。流れを実際に体験してください。

1

URLを開く

https://mcp-jade-beta.vercel.app/c/test-plc-2026 をブラウザで開く。「ナレッジOS 添削フォーム」が表示されればOK。

2

初回のみ：登録する

画面上部のタブ「はじめての方（登録）」をクリック。本名・Chatwork名・メールアドレスを入力して「登録申請する」。

申請後、田中さんが管理画面で承認しないと「添削する」タブが使えません。動作確認では「田中太郎」というすでに承認済みのユーザー名を使えば登録スキップできます。

3

「添削する」タブで原稿を貼る

「あなたの名前」に 田中太郎（承認済みユーザー名）と入れる。「何を添削する？」で種別を選ぶ（例: LP（オプトイン））。

「どのAIに食わせる？」で これから自分が使うAIを選ぶ（Claude / ChatGPT / Gemini / その他）。この情報は管理画面に記録されます。

「添削対象の原稿」欄に短い原稿を貼り付け。テスト用なので「テスト原稿: ◯◯します」みたいな短文でも OK。

4

「プロンプトを生成する」ボタンを押す

少し待つと、結果ページに切り替わって 長文のプロンプトが表示されます。これが「田中祐一として添削して」と AI に渡す指示文です。

本文には「田中祐一メソッド」「11観点」「ターゲット妥当性」などのキーワードが含まれているはず。確認してください。

5

プロンプトをコピーして AI に貼る

「プロンプトをコピー」ボタンを押すと、クリップボードに長文が入ります。それを Claude.ai / ChatGPT / Gemini のチャットに貼り付けて送信。

AI が田中祐一として添削を返してくれれば成功です。「了解しました」みたいな前置きが無く、いきなり指摘から始まれば正常。

6

添削結果を記録する（任意）

AI の出力をコピーして、画面下の「AIの出力結果をここに貼り付けて記録」欄に貼り、「記録する」ボタンを押すと、管理画面に保存されます。

動作確認では 1〜2件記録してもらえると、田中さんが管理画面で「ちゃんとデータが入っているか」確認できます。

これで Web UI 確認は完了です。 30秒〜数分の作業で終わります。

5MCP で添削する（上級者向け）

Claude.ai の MCP コネクタ機能を使うと、Web UI を開かずに、Claude.ai の会話画面から直接 「田中祐一として添削して」とリクエストできます。

5-1. Claude.ai に MCP コネクタを設定

  1. Claude.ai を開き、設定（左下の自分のアイコン）→「コネクタ」を選ぶ
  2. 「カスタムコネクタを追加」を押す
  3. 名前: 田中祐一OS、URL: https://mcp-jade-beta.vercel.app/api/mcp/g/test-plc-2026 を入れる
  4. 保存して、Claude.ai を一度リロード

5-2. Claude.ai で使う

Claude.ai の新しい会話で、コネクタが有効になっていることを確認してから、以下のように話しかける:

田中祐一として添削してください。

以下のLP原稿をお願いします:
（ここに原稿を貼る）

Claude が自動的に MCP ツール（get_correction_knowledge）を呼び出し、ナレッジを取得して、田中祐一としての添削を返してくれます。

5-3. MCPツール一覧（参考）

ツール名用途
register_user受講生登録
get_correction_knowledge添削ナレッジを取得（田中祐一として振る舞う）
log_correction_output添削結果を記録
MCP設定がうまくいかない時: Web UI で同じことができるので無理に MCP を試す必要はありません。動作確認は Web UI が完了していれば OK です。

8おまけ：田中レビュー機能

田中祐一OS の 最大の差別化ポイントです。AIが書いた添削に、田中祐一本人が後から追加コメントを書ける機能。

管理画面 → 添削ログ → セッションを開く → 画面中ほどに「田中祐一の追加コメント」エディタがあります。

3つのステータス

  • 下書き: 自分用のメモ。受講生には見えない
  • 公開: コメントとして保存（管理画面で「✦ 田中レビュー」バッジが付く）
  • ナレッジ昇格候補: 後でナレッジ本体（knowledge/）に取り込む対象としてマーク
動作確認の中で「ここ気になった」「この観点を将来ナレッジに追加したい」と感じたら、テストとして数件コメントを残してみてください。

9トラブルシューティング

「ユーザーが見つかりません」と出る
名前欄が間違っているか、未登録です。「田中太郎」で動作確認するか、登録タブから新規登録 → 田中さん（または田中さん本人）に承認してもらう必要があります。
プロンプトが長すぎてコピーできない
新しめのブラウザなら問題なくコピーできます。Chrome / Safari / Edge の最新版を推奨。Internet Explorer は非対応。
AI の出力が普通の会話っぽい（添削っぽくない）
使っているモデルが古い可能性。Claude 4 系 / GPT-5 系 / Gemini 2.5 系 を使ってください。プロンプトの先頭に「田中祐一として」と書いてあるので、AI がモード切替できれば本来は田中口調になります。
MCP コネクタが動かない
Web UI を使ってください。MCP は Claude.ai 側の仕様変更で時々不安定になります。動作確認では Web UI が動いていれば十分です。
管理画面のログインができない
パスワードを田中さんに確認してください。それでも入れない場合、Vercel 側で env が変わっている可能性があります。田中さんに連絡。
セッションが管理画面に記録されない
「記録する」ボタンを押さないと AI 出力は記録されません（プロンプト生成だけで離脱したセッションは「未完了」として記録されます）。
古い添削データが表示される
ナレッジは5分間キャッシュされています。ナレッジ更新直後に試した場合、最大5分待ってから再実行してください。
