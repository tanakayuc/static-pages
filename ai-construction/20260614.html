田中祐一AI MCP デプロイを「main git連携」へ移行 ランブック

作成日: 2026-06-14
目的: 「本番デプロイがCodexのブランチ作業ツリー由来で、mainの最新コードが本番に届かない」事故の恒久対策。Vercelをmainにgit連携し、push→自動デプロイを本当に有効化する。
担当: 田中／技術担当（Vercel管理画面・GitHub OAuth付与が必要なため）。設計: Claude。

0. 背景（なぜ必要か）

  • 2026-06-14、B4（哲学常時ロード）のコードはmainにあるのに本番に出ていなかった。
  • 原因: 本番デプロイが git連携ではなくCLI vercel --prod（vercel inspect で gitSource: None）で、しかも Codexのブランチ作業ツリー（codex/creation-overview-main等）から実行されていた。→ mainの成果が本番に届かない。
  • 詳細: メモリ project_mcp_deploy_mechanism、docs/AI進行中作業台帳.md 冒頭の警告。

1. 対象プロジェクト

  • Vercelプロジェクト: tanaka-knowledge-mcp（ID prj_ZTbbaLQoXinJX6GdXcii3DoYzXTr、team: promoter-crm）
  • 本番URL（不変・受講生/Cursor接続先）: mcp-promoter-crm.vercel.app、既定 mcp-jade-beta.vercel.app
  • アプリ実体: 71_システム開発/TanakaKnowledgeOS/apps/mcp（Next.js・Node 24.x・workspace依存なし＝自己完結）

2. 設定手順（Vercel管理画面・技術担当）

Step 1【Git連携】

  • Vercel → tanaka-knowledge-mcp → Settings → Git → Connect Git Repository
  • リポジトリ: tanakayuc/theleadpromotion（GitHub OAuth付与が必要）

Step 2【Production Branch】

  • Settings → Git → Production Branch = main

Step 3【Root Directory（最重要）】

  • Settings → Build & Deployment → Root Directory = 71_システム開発/TanakaKnowledgeOS/apps/mcp
  • ⚠️ 現在は .（CLIがapps/mcpから実行するため今は機能）。git連携はリポジトリ全体をcloneするので、. のままだとリポジトリルートをビルドしようとして壊れる。必ずアプリのパスに設定する。
  • ⚠️ パスに日本語（71_システム開発）を含む。Vercelが非ASCIIパスを正しく扱えるか、初回ビルドで必ず確認する。もし問題が出るなら、アプリを ASCII パス配下へ移すか、vercel.json/Root設定で回避を検討。

Step 4【Framework / Node】

  • Framework Preset: Next.js（自動検出）／ Node.js: 24.x（現行維持）
  • Build Command / Install Command は既定（next build / 自動）でよい

Step 5【初回デプロイ確認】

  • mainに軽微なpush（またはVercel上でRedeploy）→ git連携デプロイがReadyになるか確認
  • 完了条件:
  • vercel inspect https://mcp-promoter-crm.vercel.app で gitSource がmain/commit付きになる（CLI由来の None でなくなる）
  • 添削モードで「## 田中祐一OS 核（哲学・原理）｜常時適用」が返却ナレッジ先頭に乗る（B4が生きている）

3. 同時に必須【ブランチCLIデプロイの停止】

git連携を入れても、誰かがブランチ作業ツリーから vercel --prod を続けると本番を上書きしてB4が消える。以下を徹底:

  • Codexはブランチ作業をやめ、main直接作業に戻す（CLAUDE.mdのブランチ禁止ルール。git worktree list に出る codex/* ツリーは整理）
  • 手動 vercel --prod は原則禁止。デプロイは「mainへpush→git連携が自動デプロイ」に一本化
  • 緊急の手動デプロイが要る場合のみ、必ず git checkout main && git pull してから実行

4. 現状（2026-06-14時点）

  • B4は mainから手動デプロイ＋mcp-promoter-crm.vercel.app を手動aliasで固定して本番反映済み（実地検証OK）。
  • ただし上記git連携＋ブランチデプロイ停止が完了するまでは、ブランチデプロイで巻き戻るリスクが残る。巻き戻ったら apps/mcp（mainチェックアウト）で vercel --prod → vercel alias set <deploy> mcp-promoter-crm.vercel.app で復旧。
