PALMO × 田中祐一AI 二層データ経済圏 設計

前提資料: rules/プロジェクト責務分担表.md・rules/システム識別_PALMO_と_田中祐一AI.md・21_プロジェクト一覧/_横断_セールス/Palmo連動_検討メモ.md（未着手）

0. このドキュメントの目的

田中さんの構想「PALMOと田中祐一AIが連動してデータが行き来する巨大なナレッジ経済圏」を、境界を曖昧にせず設計する。
かつ Codex / Claude Code / Cursor のどの環境から呼んでも混乱しない実装ルールを定める。

1. 二層の役割境界（誰が何を持つか）

🟢 田中祐一AI（ナレッジ層） 🔵 PALMO（顧客データ層）
本質 田中の思考・判断の再現 顧客の事実・実績の管理
持つデータ 思考ナレッジ・添削/制作/相談の生成物・パターンDB・個人コンテキスト・5要素ナレッジ 顧客マスター・申込・入金・LTV・セールスKPI・クリエイティブ別流入・案件横断ライフサイクル
正本の場所 theleadpromotion/knowledge/ configs/ ＋ Supabase(tanaka-knowledge-os) Supabase(promoter-crm) ＋ 各案件管理/決済シート
問い 「田中ならどう判断するか」 「この顧客は誰で、何を買い、いくら払い、どうなったか」
やらない 顧客個票・入金管理 思考ナレッジ抽出・添削ノウハウ・knowledge正本管理

一文で: 田中祐一AI＝「脳」、PALMO＝「顧客台帳」。脳は判断を、台帳は事実を持つ。

3. どのツールから呼んでも混乱しない仕組み（最重要）

Codex / Claude Code / Cursor は全て同じ in-repo ドキュメントを起動時に読む。境界はそこで強制する。
