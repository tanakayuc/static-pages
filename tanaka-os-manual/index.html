1田中祐一AIの4つの機能

田中祐一AIは、単なる添削ツールではありません。PLCメンバーがプロモーションを作り、改善し、迷ったときに相談し、過去Q&Aから判断材料を探すための入口です。

Web版ではモードを選んで必要な入力欄だけを開きます。Claude.aiやClaude Codeなどを使う場合は、MCP連携によって同じ田中祐一AIのナレッジへ接続します。

Mode 01

プロモーション制作の仕組み化

LP、LINE、動画台本、セールス導線、工程表などを、工程に沿って整理しながら作るための入口です。未完成のアイデアから初稿を作り、添削で精度を上げます。

Mode 02

添削機能

LP、動画台本、オファー、セールスレター、ファネル全体などを田中祐一AIに添削してもらいます。テキストレポート、ビジュアルレポート、素材集の出力と連動します。

Mode 03

田中祐一への相談

素材が固まっていない段階の相談、前提整理、優先順位づけ、次アクションの分解に使います。いきなり制作や添削に入れないときの入口です。

Mode 04

PLCのQA検索

過去のPLC Q&Aから近い質問と回答を探し、今の悩みに近い判断材料を確認します。質問する前に、過去の回答を素早く参照するためのモードです。

最初にやること: どの機能を使う場合も、まずWeb版で会員登録をしてください。承認後に、Web UIから使うか、MCP連携でClaude側から使うかを選べます。

工程表に沿った作成モード

田中祐一AIの本命は、添削だけではありません。チャレンジローンチの工程表に沿って、コンフィグ、プロフィール、ストーリーボード、挨拶動画、LP、オファー、セールススライドなどを、上から順番に作っていく制作ナビゲーションへ拡張しています。

作成モードでは、最初から完璧な原稿を目指すのではなく、まず 65〜70点の初稿 を素早く作ります。その後、Visual Reportで Before / After を見比べながら、「田中祐一だったらこう書く」という具体例で精度を上げていきます。

重要: MCPは、ナレッジや工程表を勝手に更新するための仕組みではありません。受講生が使うMCPは、中央管理された田中祐一AIのナレッジを参照し、工程表に沿って制作・添削を進めるための入口です。
何を使うか各ステップで必要な入力素材や前工程の成果物を明示します。
何を作るかLP、動画台本、メール、スライドなど、次に出す成果物を明確にします。
なぜやるかその工程を飛ばすと後工程で何がズレるのかまで分かるようにします。
深掘り質問代表質問への回答をもとに、足りない前提を3〜5問で追加確認します。
個人前提PLCコミュニティサポート内の受講生別MD、個人コンサル、グルコン履歴をもとに、案件・進捗・過去素材を踏まえて確認できる状態を目指します。
進捗管理前回の田中フィードバック、未完了アクション、現在の工程を見て、「今の状況はこうだから次はこう」と案内します。
無料ステップ制御体験会では一部工程だけを開放し、続きは本講座側で進められる設計にします。

6MCP で使う（任意）

普段から Claude.ai を使っている方は、MCP コネクタを設定すると、Claude.ai のチャット内で田中祐一AIのナレッジを呼び出せます。登録・承認後に使ってください。

重要: MCPは2026年6月から個人MCPに切り替わりました。以前案内していた共通URL（/api/mcp/g/test-plc-2026）は使えません。必ず田中祐一AIにログインして、自分専用のMCP接続URLをコピーしてください。

6-1. 自分専用の MCP 接続URLをコピー

  1. 現在のPLC入口URL（https://tanaka-yuichi-ai.vercel.app/c/test-plc-2026）を開く
  2. 登録メールアドレスとパスワードでログインする
  3. 左側の利用者エリアに表示される「MCP接続URLをコピー」を押す
  4. コピーされるURLが、あなた専用のMCP接続URLです
個人MCPとは: 誰がMCPを使ったかを正しく記録するため、ユーザーごとに別々の接続URLを発行する仕組みです。他の人に共有せず、自分のClaude.ai / Claude Code / Cursorにだけ設定してください。

6-2. Claude.ai に MCP コネクタを追加

  1. Claude.ai を開いて、左下のアイコン（自分のアバター）をクリック
  2. 表示されるメニューから「設定」を選ぶ
  3. 設定画面の左サイドバーから「コネクタ」を選択
  4. 画面右上の「カスタムコネクタを追加」ボタンをクリック
  5. 以下を入力:
    • 名前: 田中祐一AI
    • URL: 6-1でコピーした自分専用のMCP接続URL
  6. 「保存」を押す
  7. Claude.ai を一度リロード
旧MCP URLは使えません。 以前のプロモーターCRM系ドメインや共通MCP URLを登録している場合は、ログイン後にコピーした個人MCP URLへ差し替えてください。

6-3. Claude.ai で実際に使う

新しい会話を開き、コネクタが有効になっていることを確認したうえで、こう話しかける:

田中祐一として添削してください。

以下のLP原稿をお願いします:

（ここに原稿を貼る）

Claude が自動的に MCP ツールを呼び出してナレッジを取得し、田中祐一としての添削・相談・QA検索の回答を返してくれます。

6-3. 連続して深掘りできる

MCPの強みは 同じ会話内で続けられること。最初の添削の後に:

  • 「3番目の指摘をもっと具体例つきで教えて」
  • 「修正版を書いてみたので、再度添削して」
  • 「もしオファーを ◯◯ に変えたらどう？」

と話しかけると、コンテキストを保ったまま深掘りできます。Web UIよりも往復会話しやすいので、相談や制作の壁打ちを続けたい人には MCP が向いています。

MCP の初回設定でうまくいかない場合は、無理に MCP を使わず Web UI で進めてください。Web UI と MCP は、同じ田中祐一AIのナレッジを参照する入口です。

6-4. Cursor / Claude Code でも使える？

はい。MCPに対応しているコードエディタや開発環境でも使えます。ただし、登録するURLは共通URLではなく、ログイン後にコピーした自分専用のMCP接続URLです。

https://tanaka-yuichi-ai.vercel.app/api/mcp/u/あなた専用のトークン

ただし、田中祐一AI MCPは GitHub上のナレッジ、スタティックページ、ワークフローを直接書き換えるためのものではありません。受講生向けの主用途は、中央管理されたナレッジを参照し、工程表に沿って台本・LP・オファーなどを作成、または添削することです。

更新と制作は分けます。 ソースコードやナレッジの更新は田中祐一側のGitHubアカウントと開発フローで管理し、MCPは「制作・添削・工程ナビゲート」の入口として使います。
今後の方向性: MCPはナレッジへ入る入口で、そこにAIコンシェルジュを組み合わせると、回答に応じた深掘り質問、複数のAfter案、工程ごとの初稿生成まで進められるようになります。

7実際にアウトプットしてみよう

使い方は分かったので、ここからは とにかく1回投げてみるのがおすすめです。

まずは何から使う？

  • まだ素材がないなら、相談モードで状況と目的を整理する
  • 作りたいものが決まっているなら、制作モードで工程に沿って初稿を作る
  • 今書きかけのLPがあるなら、それを貼る。途中段階でも全然OK
  • 過去に書いて反応が薄かった原稿を貼って「なぜ刺さらなかったか」を見てもらう
  • 過去Q&Aを探したいなら、QA検索モードで悩みをそのまま入力する
完璧な原稿である必要はありません。むしろ「ちょっと自信ない」「これでいいのか分からない」状態のものをそのまま投げるのが一番効果的です。

慣れてきたらこういう使い方も

  • 同じ原稿を Claude と ChatGPT 両方に投げて、出力を見比べる
  • MCP で 1セッションの中で何度も深掘りする（往復4〜5ターン）
  • 競合のLPを貼って「これに勝つには何が足りない？」と聞く
  • 添削結果を踏まえて修正版を書き直し、再度添削に投げる（PDCAを高速で回す）

このツールをどう使ってほしいか

1運用プロセスについて

このツールは、制作・添削・相談・QA検索の入口をひとつにまとめて、田中祐一の判断軸を先に通してから次の行動を決める、というプロセスを考えています。

つまり、「いきなり田中さんお願いします！」ではなく、まず田中祐一AIにかけて、制作の前提を整理したり、AIに指摘されたところを直したり、過去Q&Aで近い回答を確認してから僕のところに持ってきてほしい、ということです。これで僕の確認の質も上がるし、皆さんの原稿や企画の完成度も一段上がります。

9困ったら（FAQ）

「ユーザーが見つかりません」と出る
名前欄が登録時と違うか、まだ承認されていない可能性があります。本名で入力し直してみてください。それでもダメなら、田中さんに Chatwork で「承認待ちです」と一声かけてください。
承認の連絡は来ますか？
Chatwork ID が確認できる方には、承認時に Chatwork の個別ルームへ利用開始通知が届きます。通知が見当たらない場合でも、ポータルの各モードから普通に使えるようになっていれば承認済みです。
プロンプトが長すぎてコピーできない
新しめのブラウザなら問題なくコピーできます。Chrome / Safari / Edge の最新版を推奨。古いブラウザだと固まることがあります。
AI の添削が普通の会話っぽい（田中祐一っぽくない）
使っているモデルが古いか、別のシステムプロンプトが混ざっている可能性があります。新しい会話を開いて、もう一度プロンプトを貼ってみてください。Claude 4 系 / GPT-5 系 / Gemini 2.5 系を推奨。
MCP コネクタの設定が分からない
Web UI を使ってください。Web UI でできることは全部同じです。MCP は Claude.ai を頻繁に使う上級者向けのオプションなので、無理に設定する必要はありません。
添削結果が保存されているか不安
「記録する」ボタンを押せば管理画面に保存されます。押し忘れた場合は次回のセッションから記録してください（過去分は後から記録できません）。
同じ原稿を何度も添削していい？
全然OKです。むしろ「初版 → 修正版 → 再修正版」と何度も投げてPDCAを回すのが理想的な使い方です。
料金はかかる？
田中祐一AI自体は無料です。ただし、AI（Claude / ChatGPT / Gemini）は皆さん自身のサブスク（無料枠 or 有料プラン）で動きます。長文の添削は Claude Pro / ChatGPT Plus / Gemini Advanced など有料プランがあるとスムーズです。
URLが急に使えなくなった
運用上の理由でURLを変更・無効化する場合があります。Chatwork で新しいURLが配布されますので、そちらに切り替えてください。
競合や外部の人にURLを知られたら？
あなた専用URLが第三者に渡っても、その人が田中さんから承認されないと使えません（名前で本人確認するため）。気になる場合は田中さんに連絡を。
