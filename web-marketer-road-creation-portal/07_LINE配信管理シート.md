# LINE配信管理シート

## 目的

LINEオープンチャット、課題キーワード用LINE、公式LINEを混同せず、配信コンテンツと事務連絡を分けて管理する。

## チャネル定義

| チャネル | 役割 | 主な内容 |
| --- | --- | --- |
| LINEオープンチャット | 5日間チャレンジ本体 | ライブ案内、課題、Q&A、コミュニティ運営 |
| 課題キーワード用LINE | 課題提出後の特典受け取り | キーワード送信、特典配布、自動返信 |
| 公式LINE | 販売期のプッシュ | セールスページ案内、締切、購入促進 |

## 配信タイプ

| タイプ | 内容 | 作り込み方 |
| --- | --- | --- |
| 事務的連絡 | 開始時間、URL、課題締切、アーカイブ、募集終了 | テンプレート化し、正確性を優先 |
| ルーティン配信 | Day前後の教育、リマインド、販売プッシュ | コンセプトに沿って作り込む |
| Q&A/個別対応 | 質問回答、追加補足、参加者への個別返信 | ログとして残し、ナレッジ抽出対象にする |

## フェーズ別配信

| フェーズ | 配信の目的 | 主な配信 |
| --- | --- | --- |
| ライブ前 | 期待値形成、参加準備、視聴率向上 | 歓迎、企画意図、事前学習、開始リマインド |
| 価値提供中 | ライブ視聴、課題提出、理解促進 | Day案内、アーカイブ、課題、特典、補足回答 |
| セールス前 | 販売への橋渡し | 実践環境の必要性、初実績の壁、予告 |
| セールス中 | セールスページ閲覧、購入判断 | 販売開始、参加者状況、締切、よくある質問 |
| 販売終了後 | 締切終了、次回導線、参加者フォロー | 募集終了、購入者案内、未購入者フォロー |

## ルール

- 事務連絡は短く、迷わせない
- 教育配信はコンセプトの認識変化に沿って書く
- Q&Aは消さずにログ化する
- 反応が高かった投稿は次回プロモーションのナレッジ候補にする
- 公式LINEはセールス期に絞り、オープンチャットの熱量と連動させる

## 実運用で必要な列

| 列名 | 用途 |
| --- | --- |
| 配信日/時刻 | いつ送るか |
| フェーズ | ライブ前、価値提供中、セールス中など |
| チャネル | オープンチャット、課題LINE、公式LINE、メルマガ |
| 配信タイプ | 事務連絡、定期配信、Q&A、販売、特典配布 |
| 対象者 | 全員、未視聴者、課題未提出者、購入者、未購入者 |
| 目的 | 視聴促進、課題提出、販売ページ遷移、購入促進など |
| 本文リンク | Google Docs、Markdown、実ログ |
| CTA | 視聴、提出、返信、購入、確認 |
| 自動化 | 手動、予約配信、自動返信、除外条件あり |
| ステータス | 下書き、予約済み、配信済み、差し替え要 |

## 配信分類の追加タグ

| タグ | 内容 |
| --- | --- |
| 課題提出促進 | 課題の提出率を上げる |
| 特典配布 | 課題提出後の特典受け取り |
| 販売予告 | セールス開始前の期待値形成 |
| 販売開始 | セールスページ公開 |
| 締切前 | 緊急性と限定性を伝える |
| 購入者案内 | 購入後の参加案内 |
| 未購入者フォロー | 販売終了後の次回導線 |
