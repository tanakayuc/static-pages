## HTMLページのURL分割設計

| 分類 | 役割 | 主な内容 |
| --- | --- | --- |
| レポート | 案件全体を把握する | 全体構成、工程表 |

## URL分割ルール

HTMLページは、添削レポートやセールスレポートと同じく、情報量を1ページに集約しすぎない構成を標準とします。
ユーザーはサイドバーから必要なURLへ移動し、各ページでその役割だけを確認します。

| URL | 役割 |
| --- | --- |
| `roadmap.html` | 新規制作モードの本体。工程番号を基準に進行する |
| `visual-report.html` | ファネル全体、必要素材、VSL配置、KPI、導線の全体像を見る |
| `assets.html` | 制作物を集客・価値提供・販売の3階層で見る詳細ページ |
| `target.html` / `research.html` / `concept.html` / `config.html` / `offer.html` | 各設計シートの詳細を見る |
| `lp.html` / `value.html` / `sales-page.html` | 集客・価値提供・販売の制作物カテゴリを見る |
| `optin-after-mails.html` / `traffic-mails.html` / `line.html` / `sales-mails.html` / `sales-line.html` | メール/LINEをフェーズ別に見る |
| `head.html` / `sales-head.html` / `purchase-complete.html` | 個別制作物を見る。KPIは全体構成へ統合する |

## サイドバー設計の注意

- 上部の重複リンクは置かず、移動はサイドバーに集約します。
- サイドバーのレポート欄は「全体構成」「工程表」に絞ります。
- 設計シートや制作物ページは、工程表から入る詳細ページとして扱います。
- 制作物ページは、工程順ではなく「集客」「価値提供」「販売」の3階層で分類します。
- 個別ページでは、実際の原本URL、素材ファイル、未確定事項を明記します。
