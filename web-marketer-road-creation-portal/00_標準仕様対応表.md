## URL分割ルール

制作ポータルは、添削レポートやセールスレポートと同じく、情報量を1ページに集約しすぎない構成を標準とします。
ユーザーはサイドバーから必要なURLへ移動し、各ページでその役割だけを確認します。

| URL | 役割 |
| --- | --- |
| `index.html` | 案件の入口。相手名、案件名、全体像だけを見る |
| `roadmap.html` | 工程表を見る |
| `sheets.html` | 各種設計シートの親ページを見る |
| `assets.html` | 制作物の親ページを見る |
| `stepmail.html` | ステップメールの一覧を見る |
| `concept.html` / `research.html` / `offer.html` | 各シートの詳細を見る |
| `head.html` / `line.html` / `kpi.html` / `files.html` | 個別制作物、KPI、原本管理を見る |

## サイドバー設計の注意

- 上部の重複リンクは置かず、移動はサイドバーに集約します。
- サイドバーは「レポート」「設計シート」「制作物」の3分類にします。
- 個別ページでは、実際の原本URL、MDパス、未確定事項を明記します。
