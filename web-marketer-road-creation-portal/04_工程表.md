# 工程表

この工程表は、細かな操作手順ではなく「何を決め、何が完成するか」を示すためのものです。制作モードでは、各工程の成果物が素材集に積み上がっていきます。

## ファネル全体

```
オプトインLP
  -> LINEオープンチャット
  -> 5日間ライブ / 課題
  -> セールスページ
  -> 決済 / 購入完了ページ
```

## 工程一覧

| 工程 | 目的 | 完成アウトプット | 状態 |
| --- | --- | --- | --- |
| 1. 全体設計 | 商品、販売方式、ファネル、KPIの仮決め | 全体設計メモ、KPI仮シミュレーション | サンプル化対象 |
| 2. コンセプト構築 | ターゲット、ライバル、自社強み、ポジションを固める | コンセプトシート、リサーチシート、プロフィール、コンフィグ | 作成済みサンプル |
| 3. オファー設計 | 本命商品の約束、提供内容、特典、限定性を整理する | オファーシート、購入障壁整理 | 作成済みサンプル |
| 4. 集客ページ制作 | 登録させるための訴求とヘッドを作る | オプトインLP、ヘッドデザイン指示書、登録経路別URL | 原本あり |
| 5. 登録後導線 | 登録直後の行動を明確にする | サンキューページ、LINEオープンチャット、固定ノート | 原本あり |
| 6. 価値提供 | 5日間で認識を変え、課題で参加度を上げる | Day1〜Day5ライブ台本、動画、課題、特典 | 原本あり |
| 7. 配信運用 | 参加者に必要な案内と教育配信を届ける | オープンチャット配信、メルマガ、公式LINE | 原本あり |
| 8. セールス | 価値提供後に本命商品を販売する | セールスページ、販売LINE、販売メルマガ、決済導線 | 原本あり |
| 9. 購入後/改善 | 購入者を受け入れ、次回改善に使う | 購入完了ページ、振り返り、改善メモ | 一部原本あり |

## 画面で見せるべき粒度

ユーザーに見せる工程表では、細かいスキル名や作業手順は省く。各工程に「何を作るか」と「完成物」が表示され、クリックすると該当の素材やレポートに移動できれば十分。

## 制作モード内での扱い

- 全体設計が完了したら、この工程表と制作物一覧を生成する
- コンセプト工程が進むと、コンセプトシート、リサーチシート、プロフィール、コンフィグが更新される
- オファー工程が進むと、オファーシートが更新される
- LPや動画台本などの個別成果物は、工程表から該当ファイルへリンクする
- 原本差し替えが起きた場合は、該当Markdownを更新し、HTMLポータルを再生成する

## KPI仮シミュレーション

直販型チャレンジローンチのサンプル値です。実績値ではなく、全体規模を把握するための仮置きです。

| 指標 | 仮値 | 算出 |
| --- | --- | --- |
| 目標購入数 | 30件 | 販売終了後ログの参加者数をサンプル基準にする |
| 商品単価 | 60,000円 | サンプル仮値 |
| 目標売上 | 1,800,000円 | 30件 x 60,000円 |
| オプトインLP成約率 | 10% | 入口LPの仮基準値 |
| チャレンジ有効参加率 | 60% | LINE登録後にライブ/課題へ動く割合 |
| セールスページ遷移率 | 80% | 有効参加者のうち販売ページを見る割合 |
| セールスページ購入率 | 12.5% | 240アクセス中30件購入の想定 |
| 必要オプトイン数 | 500件 | 30件 / 60% / 80% / 12.5% |
| 必要アクセス数 | 5,000アクセス | 500件 / 10% |

このKPIは「ハウスリスト中心の直販チャレンジ」の仮値です。広告運用をする場合はCPAを別途設定し、必要オプトイン数から広告予算を逆算します。
